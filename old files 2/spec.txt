SYSTEM DESIGN GOALS
1. Take data on utility functions of several regions (currently four) with respect to COPPER
DOMINATED PRODUCT and ALUMINUM DOMINATED PRODUCT.
2. Expected input is outlined below. Three datasets.
3. The expected output are the following:
    - tau values per region
    - graphs per region of the market share of copper where the following variable is varied
        - copper price
        - aluminum price
        - ratio of copper to aluminum price

EXPECTED INPUT
there are 3 expected input data lists in tiers: global data, regional data, and product data.

1. GLOBAL DATA (global_data)
This is data TRUE ACROSS ALL REGIONS. The following fields are expected.

"attribute_1": "total_cost", # THIS IS ALWAYS ATTRIBUTE 1
"attribute_2": "material_continuity", # THIS IS ALWAYS ATTRIBUTE 2
"attribute_3": "electrical_performance", # CAN CHANGE
"attribute_4": "mechanical_integrity", # CAN CHANGE
"attribute_5": "thermal_performance", # CAN CHANGE
"direction_attribute_1": negative, # THIS CAN ONLY BE EITHER "positive" or "negative". "positive": means more is better. "negative" means less is better
"direction_attribute_2": negative,
"direction_attribute_3": negative,
"direction_attribute_4": negative,
"direction_attribute_5": negative,
"attribute_1_min": 25.00, # MAKE SURE THE COST MATCHES FOR PRODUCT COST
"attribute_1_max": 45.00,
"attribute_2_min": 0.80,
"attribute_2_max": 0.95,
"attribute_3_min": 0.65,
"attribute_3_max": 0.85,
"attribute_4_min": 0.60,
"attribute_4_max": 0.80,
"attribute_5_min": 0.55,
"attribute_5_max": 0.75
